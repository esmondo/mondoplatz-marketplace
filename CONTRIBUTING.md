# Contributing to Mondoplatz

Thank you for your interest in contributing to Mondoplatz! This marketplace thrives on community contributions.

## Ways to Contribute

### 1. Adding New Plugins

Want to share your Claude Code plugin with the community?

1. Read the [Plugin Development Guidelines](./docs/PLUGIN_GUIDELINES.md)
2. Follow the standard plugin structure
3. Test thoroughly
4. Submit a PR

### 2. Improving Existing Plugins

Found a bug or want to enhance an existing plugin?

1. Open an issue describing the improvement
2. Wait for maintainer feedback
3. Make your changes in a fork
4. Submit a PR

### 3. Documentation

Help improve documentation:
- Fix typos or unclear explanations
- Add examples and use cases
- Translate documentation
- Create tutorials

## Pull Request Process

1. **Fork** the repository
2. **Create a branch**: `git checkout -b feature/your-feature-name`
3. **Make your changes**
4. **Test thoroughly**
5. **Commit** with clear messages: `git commit -m "Add: brief description"`
6. **Push** to your fork: `git push origin feature/your-feature-name`
7. **Open a Pull Request**

### PR Guidelines

- Keep PRs focused on a single feature/fix
- Write clear, descriptive PR titles
- Include a detailed description of changes
- Link related issues
- Ensure all tests pass
- Update documentation as needed

## Code Style

- Follow existing code conventions
- Use clear, descriptive variable names
- Comment complex logic
- Keep functions small and focused

## Plugin Submission Checklist

- [ ] Plugin follows standard structure
- [ ] `plugin.json` has all required fields
- [ ] README.md is comprehensive
- [ ] All commands/skills are tested
- [ ] Added to marketplace.json
- [ ] Added to main README.md
- [ ] No security vulnerabilities

## Questions?

Open an issue with the `question` label, and we'll help you out!
